Creating a Workshop Website

February 2015

Hello, and welcome to Software Carpentry.

What

Create your Software Carpentry workshop website

https://github.com/swcarpentry/workshop-template
This quick tutorial will show you how to create a website for a Software Carpentry workshop using Git, GitHub, and a bit of HTML.

Why

  1. Allows instructors to update when they need to
  2. Permits data to be accessible before, during, and after workshop
  3. Ensures at least one instructor can use Git and GitHub
We create a small website for each workshop on GitHub for three reasons. First, it allows instructors to update their website when they need to without needing access to the main Software Carpentry site. Second, instructors can put whatever information they want on their website and be sure it will be available before and after the workshop. And third, doing this ensures that at least one instructors can use Git and GitHub.

Overview

  • Create your website in a GitHub repository:
    • Repo: https://github.com/username/workshop
    • Branch: gh-pages
  • GitHub automatically turns this into a website:
    • Site: http://username.github.io/workshop/
  • Plain pages are published as-is.
  • Pages with special headers are compiled.
Here's how it works. If you create a repository at https://github.com/username/workshop that has a branch called gh-pages, GitHub automatically creates a website at http://username.github.io/workshop/. Plain old HTML pages in that repository are published as-is, but Markdown and HTML pages that have a special header are compiled.

Behind the Scenes

  • GitHub uses Jekyll to compile websites
  • Combines values from page's header...
  • ...and from a file called _config.yml...
  • ...with page templates stored in _layouts...
  • ...and snippets of HTML in _includes...
  • ...to create finished HTML pages
Behind the scenes, GitHub uses a program called Jekyll to compile websites. It combines values from the page's header, values from a configuration file, page templates stored in a _layouts directory, and bits of HTML stored in an _includes directory to create finished HTML pages.

Behind the Scenes

Naming

  • Repositories are named YYYY-MM-DD-site 
    https://github.com/gvwilson/2015-07-01-miskatonic
  • Website has a predictable name 
    http://gvwilson.github.io/2015-07-01-miskatonic
By convention, we use names like "YYYY dash MM dash DD dash site" for workshop repositories, where "YYYY dash MM dash DD" is the start date and "site" is a short name for the host institution. This repository is usually created and owned by the lead instructor (in this case, "gvwilson"). The published website's name is then "username dot github dot io slash workshop name".

Content

  • Put anything in the website
  • But index.html must contain metadata needed by main website
You can add anything you want to your workshop's website. However, the header of the home page, index.html, must contain certain values in order for us to list your workshop on the main Software Carpentry website.

There's a Catch

  • A user can only have one fork of a repo
  • But instructors teach many workshops
  • So we cannot simply fork workshop-template
  • Must import the template into a new repo
Now, in an ideal world the lead instructor would create a workshop website by forking the workshop-template repository. Unfortunately, though, GitHub only allows a user to have one fork of any particular repository. Since instructors may teach many workshops, they can't simply fork the tepmlate. Instead, we must create an empty repository and then import the template's contents into it.

Step 1: Import

  1. Go to http://import.github.com
  2. Enter this URL: 
    https://github.com/swcarpentry/workshop-template.git
  3. Press the button to check the URL (GitHub won't import until you do)
  4. Select the owner and name
  5. Make sure the new repository is public
The first step is to import the workshop template. To do this, go to import.github.com and enter the URL of the workshop template repository (with "dot git" on the end). Press the button to check the URL, then set the owner and name and make sure the new repository will be public.

Step 2: Customize

  • Go to your newly-created repository
  • Edit the header of index.html
  • Edit _config.yml to set lesson_repo and lesson_site
  • You can edit on GitHub or by cloning and editing on your desktop
Once GitHub completes the import, go to your newly-created repository and edit the two files "index.html" and "_config.yml". "index.html" is the home page for your repository; the values in its header specify the date and location of your workshop, the names of the instructors, and so on. "_config.yml" is the Jekyll configuration file. The only two values you need to change here are "lesson_repo" and "lesson_site". You can edit these files directly on GitHub if you want, or by cloning the repository to your desktop and editing there, whichever you're most comfortable with.

Step 3: Check

  • Run tools/check.py to validate...
  • ...tools/preview to preview locally...
  • ...or tools/clean to clean up
Once you have edited these files, though, you will want to clone the repository to your desktop so that you can run "tools/check.py" to make sure everything's OK. If you have Jekyll installed, you can also run "tools/preview" to preview changes to your website locally, and then "tools/clean" to clean up the files that previewing creates.

4. Publish

  • Push to website
    • If you didn't simply edit there
  • Check your changes
  • Mail URL to admin@software-carpentry.org
  • Or create pull request for https://github.com/swcarpentry/site that updates config/workshop_urls.yml.
Finally, if you didn't edit directly on GitHub, please remember to push your changes to your repository's gh-pages branch. GitHub will publish those changes ten to fifteen seconds later. Once you're sure the site is OK, please send the repository's URL to the Software Carpentry administrator to be added to the main website (or just submit a pull request to the main website's repository that adds your workshop to config/workshop_urls.yml).

Improving the Template

  • Fork workshop-template and send pull requests
  • Or file issues
If you'd like to help us improve the template or its documentation, please fork the workshop-template repository and send us a pull request, or file an issue in that repository for us to look at.
